# 租户管理与登录

低代码平台支持多租户能力，平台运营方可通过创建不同租户来服务多个业务域或子公司，实现资源隔离与权限独立。


## 一、平台租户模型说明

低代码引擎采用租户 → 组织 → 用户的典型三层结构：

```
平台（统一入口）
├── 租户A
│ ├── 组织结构A-1
│ └── 用户群A
└── 租户B
├── 组织结构B-1
└── 用户群B
```

- **租户**：独立业务空间，资源隔离；
- **组织**：租户内部的部门结构；
- **用户**：挂载组织的账号，绑定角色登录平台；


## 二、租户创建方式

平台支持两种部署方式：

- 🏢 公有部署：不支持自助创建，需平台管理员审核创建；
- 🏠 私有部署：支持通过「租户管理」后台创建租户信息；

📷 创建租户：

![租户创建](../images/tenant/tenant-create.png)

创建后平台将自动初始化：
- 默认管理员账号（`[租户编码]@admin`）；
- 系统内置角色；
- 默认组织节点；

📷 租户管理页面：

![租户列表](../images/tenant/tenant-list.png)

## 五、常见问题

### Q：一个用户可以属于多个租户吗？
A：不支持，账号在租户内唯一。

### Q：租户是否可以删除？
A：支持，但需先清除其下属数据。

### Q：如何重置密码？
A：联系管理员在后台重置，平台不提供找回入口。
